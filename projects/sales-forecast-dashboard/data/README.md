# Data Folder

This folder contains all datasets used for the Sales Performance & Forecasting Dashboard project.

---

## 📁 Data Files

| File | Description | Source | Size |
|------|-------------|--------|------|
| `superstore_sales.csv` | Raw Superstore sales data (2015-2018) | Kaggle | ~9,994 rows |
| `cleaned_sales_data.csv` | Cleaned and processed data (after EDA) | Generated by notebook | ~9,994 rows |

---

## 📊 Dataset Details

### superstore_sales.csv (Raw Data)

**Source**: [Superstore Sales Dataset on Kaggle](https://www.kaggle.com/datasets/rohitsahoo/sales-forecasting)

**Description**: This is a classic retail dataset containing sales transactions from a fictional Superstore.

**Columns**:
| Column | Description |
|--------|-------------|
| `Row ID` | Unique row identifier |
| `Order ID` | Unique order identifier |
| `Order Date` | Date of purchase (DD/MM/YYYY) |
| `Ship Date` | Date of shipping |
| `Ship Mode` | Shipping method (e.g., Standard, Second Class) |
| `Customer ID` | Unique customer identifier |
| `Customer Name` | Customer full name |
| `Segment` | Customer segment (Consumer, Corporate, Home Office) |
| `Country` | Customer country |
| `City` | Customer city |
| `State` | Customer state |
| `Postal Code` | Customer postal code |
| `Region` | Customer region (North, South, East, West) |
| `Product ID` | Unique product identifier |
| `Category` | Product category (Furniture, Office Supplies, Technology) |
| `Sub-Category` | Product sub-category |
| `Product Name` | Product description |
| `Sales` | Total sales amount ($) |

**Key Characteristics**:
- 📅 Date Range: 2015-01-13 to 2018-12-25
- 📊 Total Records: 9,994 rows
- 🏢 Regions: 4 (North, South, East, West)
- 📦 Categories: 3 (Furniture, Office Supplies, Technology)
- 👥 Customer Segments: 3 (Consumer, Corporate, Home Office)

**⚠️ Important Note**: This dataset does NOT include a `Profit` column. The version used here contains only `Sales` data.

### cleaned_sales_data.csv (Processed Data)

**Generated by**: `01_eda_sales_data.ipynb`

**Description**: The raw data after cleaning, transformation, and standardization.

**Processing Steps**:
1. ✅ Column names standardized (removed spaces, consistent capitalization)
2. ✅ `Order Date` converted to datetime format
3. ✅ Missing values handled (dropped or imputed)
4. ✅ Data validated and filtered for quality

**Why Use This File**:
- Ready for Tableau/Power BI dashboards
- Consistent column names for easy analysis
- Cleaned and validated for accuracy

---

### sales_forecast_3month.csv (Forecast Results)

**Generated by**: `02_sales_forecast.ipynb`

**Description**: Facebook Prophet forecast outputs for 3 months (November 2018 – January 2019).

**Key Columns**:
| Column | Description |
|--------|-------------|
| `ds` | Date (monthly timestamps) |
| `yhat` | Forecasted sales ($) |
| `yhat_lower` | Lower bound (80% confidence interval) |
| `yhat_upper` | Upper bound (80% confidence interval) |
| `trend` | Underlying trend component |
| `additive_terms` | Seasonal components |

**Forecast Summary**:
- 📈 Total Forecasted Sales (3 months): **$305.48**
- 📊 Average Monthly Sales: **$101.83**
- 📅 Peak Month: **November 2018** ($127.70)
- 📉 Lowest Month: **January 2019** ($75.67)

**⚠️ Data Limitations**:
> This forecast is based on historical data from 2015-2018 (only ~3 years). Yearly seasonality could not be reliably detected with this limited data. Forecast horizon is limited to 3 months for reliability.

## 🚀 How to Use This Data

### For Analysis (EDA & Forecasting)
1. Use `cleaned_sales_data.csv` in your Python notebooks.
2. It's already processed and ready for analysis.
3. Run `01_eda_sales_data.ipynb` to regenerate from raw data.

### For Dashboard (Tableau / Power BI)
1. Connect Tableau/Power BI to `cleaned_sales_data.csv`.
2. Build visualizations using the clean columns.
3. Dashboard ready!

### For Forecasting
1. Use `02_sales_forecast.ipynb` to regenerate forecasts.
2. Load `sales_forecast_3month.csv` for dashboard integration.
3. Update forecast monthly as new data becomes available.
